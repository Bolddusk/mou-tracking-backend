# MOU Change Logs — Frontend Integration

Audit trail for every MOU (proposal): who changed what, with before/after values.

---

| Role | Dashboard Change History page |
|------|------------------------------|
| **Super Admin / Admin** | `GET /change-logs/recent` + filters (see below) |
| **Sector Lead** | `GET /change-logs/sector` — all changes on MOUs in **their assigned sectors** + filters |
| **Party A / others** | `GET /change-logs/mine` — only changes they made |

Load filter dropdowns once: `GET /change-logs/filter-options`

---

## Filtering (all list endpoints)

### Filter options (dropdowns)

```
GET /api/proposals/change-logs/filter-options
```

**Super Admin / Admin** — full lists. **Sector Lead** — only their jurisdiction sectors + sector leads in those sectors.

```json
{
  "viewer_role": "super_admin",
  "sector_leads": [
    { "id": 33, "full_name": "Engr. Muhammad Asif, CEWRI-PARC", "email": "...", "sectors": ["Smart Farming Solutions"] }
  ],
  "sectors": ["Smart Farming Solutions", "Seed Production"],
  "mou_statuses": [
    { "value": "active", "label": "Active" },
    { "value": "inactive", "label": "Inactive" },
    { "value": "execution", "label": "Execution" }
  ],
  "changed_by_roles": [
    { "value": "super_admin", "label": "Super Admin" },
    { "value": "sector_lead", "label": "Sector Lead" }
  ],
  "scoped_sectors": null
}
```

**Removed from UI/API:** `action`, `proposal_id` / `mou_id`, workflow statuses (`draft`, `approved`, etc.)

For sector lead, `scoped_sectors` is their assigned sector list.

---

### Query parameters (combine as needed)

| Param | Applies to | Description |
|-------|------------|-------------|
| `q` or `search` | All list APIs | MOU company, MOU ID in search text, sector name, changer name |
| `sector_lead_id` | **Super Admin** `/recent` | Changes made **by** this sector lead |
| `changed_by` | Super Admin `/recent`, Sector `/sector` | Changes by any user id |
| `changed_by_role` / `role` | Super Admin, Sector | e.g. `sector_lead`, `super_admin`, `party_a` |
| `sector` | Super Admin, Sector | MOU sector (sector lead: must be in their jurisdiction) |
| `mou_status` | All | MOU lifecycle: `active`, `inactive`, `execution` |
| `from` / `date_from` | All | Start date `YYYY-MM-DD` or ISO |
| `to` / `date_to` | All | End date |
| `limit` / `offset` | All | Pagination (max 200) |
| `mine_only` | **Sector Lead** `/sector` | `true` = only their own edits (still within jurisdiction MOUs) |

```
GET /change-logs/recent?sector_lead_id=33

GET /change-logs/recent?from=2026-07-01&to=2026-07-06&changed_by_role=sector_lead

### Sector Lead examples

```
GET /change-logs/sector

## Download report

### Per-MOU export (MOU detail → Change History tab)

```
GET /api/proposals/:id/change-logs/export?format=csv
GET /api/proposals/:id/change-logs/export?format=xlsx
```

| Item | Detail |
|------|--------|
| Auth | Same as `GET /api/proposals/:id/change-logs` |
| Scope | Super Admin / Admin → all changes on that MOU; others → only their own |
| Filename | `mou-{id}-change-logs-2026-07-06.csv` |
| Columns | Log ID, Date/Time, MOU ID, MOU, Sector, MOU Status, Changed By, Role, Action, Summary, Fields Changed, Before → After |

```js
fetch(`/api/proposals/${proposalId}/change-logs/export?format=xlsx`, {
  headers: { Authorization: `Bearer ${token}` },
});
```

---

### Global export (Change Logs dashboard)

**Same filters as the on-screen list** — pass the current filter query params to export. If no filters are set, you get a **general** report for your role scope.

```
GET /api/proposals/change-logs/export?format=csv
GET /api/proposals/change-logs/export?format=xlsx
GET /api/proposals/change-logs/export?format=csv&proposal_id=318
GET /api/proposals/change-logs/export?format=csv&sector_lead_id=33&mou_status=active&from=2026-07-01
```

Optional `proposal_id` / `mou_id` on global export to download one MOU from the dashboard.

| Role | Export scope |
|------|----------------|
| Super Admin / Admin | All change logs (optional filters) |
| Sector Lead | MOUs in assigned sectors (optional filters + `mine_only`) |
| Party A / others | Only their own changes |

**Formats:** `csv` (default), `xlsx`

**Filename:**
- No filters → `mou-change-logs-general-2026-07-06.csv`
- With filters → `mou-change-logs-filtered-2026-07-06.xlsx`

**Report contents:**
- Header sheet / rows: applied filters, generated by, total count
- Data columns: Log ID, Date, MOU ID, MOU, Sector, MOU Status, Changed By, Role, Action, Summary, Fields Changed, Before → After

**Frontend:** On "Download Report" click, call export with **the same query string** as the current list API.

```js
// Example — mirror list filters
const params = new URLSearchParams({
  format: 'xlsx',
  ...(search && { q: search }),
  ...(sectorLeadId && { sector_lead_id: sectorLeadId }),
  ...(sector && { sector }),
  ...(mouStatus && { mou_status: mouStatus }),
  ...(role && { changed_by_role: role }),
  ...(from && { from }),
  ...(to && { to }),
});
window.location.href = `/api/proposals/change-logs/export?${params}`;
// Or fetch with Authorization header + blob download
```

## APIs

### 1. Latest changes feed (Super Admin / Admin — default page load)

```
GET /api/proposals/change-logs/recent
GET /api/proposals/change-logs/recent?limit=50&offset=0
GET /api/proposals/change-logs/recent?q=grain
GET /api/proposals/change-logs/recent?proposal_id=318
```

**No MOU selection required.** Returns the newest change-log entries across every MOU, newest first.

## Logged actions

| `action` | When |
|----------|------|
| `created` | New draft MOU created |
| `draft_saved` | Draft saved with field changes |
| `submitted` | Submitted for sector lead review |
| `resubmitted` | Rejected MOU resubmitted |
| `fields_updated` | `PATCH /api/proposals/:id/fields` |
| `party_contacts_updated` | `PATCH /api/proposals/:id/party-contacts` |
| `mou_updated` | MOU text fields updated |
| `mou_file_uploaded` | New MOU PDF uploaded |
| `mou_acknowledged` | Party A or B acknowledged MOU |
| `approved` | Sector lead approved |
| `rejected` | Sector lead rejected |
| `deal_closed` | Deal closed |

## UI suggestions

### Super Admin / Admin — Change History page

1. **On page load:** `GET /change-logs/recent?limit=50` — show timeline immediately (no dropdown step)
2. **Search box (optional):** same endpoint with `?q=...` or `?proposal_id=318`
3. MOU dropdown is **optional** — only if you want a dedicated single-MOU filter
4. Each row: MOU label (`proposal_label`) + link to `/proposals/:id`, changer, action, before/after

### Sector lead / Party A / others — Change History page

1. **Do not** use the admin MOU dropdown
2. Load `GET /change-logs/mine` — timeline of their own edits
3. On MOU detail **Change History** tab: same filter (`viewer_scope: own`)

### MOU detail tab & timeline rows

Each log entry **must show field-level before/after** — not just action title.

Use `changes[]` (or ready-made `change_details[]` strings):

| API field | Use |
|-----------|-----|
| `changed_by_name` | Who made the change |
| `action_label` | What type of action (edit, approve, etc.) |
| `changes[].label` | Human field name |
| `changes[].old_value` | Value **before** (show `—` if null) |
| `changes[].new_value` | Value **after** (show `—` if null) |
| `changes[].display` | Pre-formatted: `"Sector: X → Y"` |
| `change_details` | Array of `display` strings — easy for bullet list |
| `fields_changed` | Quick chips: `["Sector", "Progress"]` |

**Example row UI:**

```
Engr. Muhammad Asif · MOU fields updated · 2 hours ago
MOU #318 — The Grain Industries / Dongying Fengtu...

  Sector          Seed Production  →  Smart Farming Solutions
  Progress        Meeting scheduled  →  Site visit completed
```

Expand/collapse optional — but **always list only the fields that actually changed** (`changes` array), never the whole MOU.

- Super Admin / Admin: all changes on timeline
- Others: subtitle *"Showing changes you made"* when `viewer_scope === 'own'`

---

## Notes

- Logs are **append-only** (no edit/delete API)
- `old_value` / `new_value` may be `null` when field was empty
- Long JSON values are truncated server-side (~2000 chars)
- Imported historic MOUs have **no retroactive logs** — only changes after deploy
- **Auto-synced fields are hidden** from logs: when Pakistani Company or Chinese Venture changes, internal `proposal_title` sync is not logged separately; when Conference dropdown changes, `conference_name` / `conference_info` sub-fields collapse into the conference change
